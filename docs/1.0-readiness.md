# 1.0 Readiness

BlastCSS can move to `1.0.0` when these release gates pass and the maintainers accept the public API as stable.

## Required Gates

- `npm run test:static`
- `npm run test:browser`
- `npm pack --dry-run`
- Size budgets stay under configured limits.
- Browser tests pass in Chromium, Firefox, WebKit, mobile Chrome, and mobile Safari profiles.
- Axe scans pass for docs, examples, and benchmark fixtures.
- Visual baselines are reviewed for docs, examples, and benchmark fixtures.
- Public API review is complete.
- Changelog and migration notes are updated.

## API Freeze Checklist

- Package exports are final.
- `b-` component names are final.
- Utility class names are final.
- Token names are final.
- Cascade layer names are final.
- Browser support policy is final.
- Optional JS helper API is final.

## Release Decision

The project should only publish `1.0.0` after at least one real application has been built with it and any painful naming, theming, accessibility, or component gaps have been addressed.
