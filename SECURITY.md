# Security Policy

BlastCSS is primarily a CSS package and does not process user data at runtime.

## Supported Versions

Security fixes are provided for the latest minor version until the project reaches `1.0.0`.

## Reporting Issues

Report security issues privately to the project maintainers. Do not publish exploit details until a fix is available.

## Scope

Relevant security issues include:

- Supply-chain compromise.
- Malicious package content.
- Build or release workflow vulnerabilities.
- CSS that enables misleading or unsafe UI behavior in documented components.
